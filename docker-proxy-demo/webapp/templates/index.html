💡 What's Happening Here?

Reverse Proxy role: NGINX is the single public entrypoint and forwards requests by path (`/webapp-api`, `/internal-api/*`) while adding forwarding headers.

Load Balancer role: The `/` and `/webapp-api` routes are distributed across 3 web app instances with round-robin.

Internal Network role: The internal API has no host port published; it is reachable externally only through NGINX proxy routes.
