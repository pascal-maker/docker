# Execute Plan Workflow

Use this workflow when carrying a phase plan from implementation through
verification.

## Steps

1. Read the plan objective, context, interfaces, and acceptance criteria before
   making changes.
2. Implement the scoped tasks without widening the plan unless the codebase
   forces it.
3. Run the validation commands listed in the plan as you complete each task or
   wave.
4. Stop and capture blockers instead of suppressing typing, lint, or test
   failures.
5. Finish by writing a concise summary that covers what changed, what was
   verified, and any remaining risks.
