<objective>
Extend the ts-morph bridge to detect `this.forceUpdate()` call sites and implement the deterministic classifier layer: hard-block rules that unconditionally route unmigrateable components to manual tier, and heuristic scoring that assigns preliminary tiers based on CLASS-04 criteria.

Purpose: The deterministic classifier layer runs before any LLM call, catching all unconditionally-manual components (no AI judgment needed) and pre-classifying simple components. This reduces LLM cost and ensures blocklist decisions are never wrong.

Output: Updated `react.ts` with `hasForceUpdate` detection, `ts_morph_engine.py` with field mapping, `classifier.py` with blocklist and heuristic functions, and comprehensive unit tests.
</objective>

    1. Add `from __future__ import annotations` and imports:

       ```python
       _HARD_BLOCK_LIFECYCLE_METHODS: frozenset[str] = frozenset({
           "getSnapshotBeforeUpdate",
           "componentDidCatch",
           "getDerivedStateFromError",
           "getDerivedStateFromProps",
       })
       ```

    3. Create `apply_hard_blocks(info: ComponentInfo) -> ComponentClassification | None`:
       - Check `set(info.lifecycle_methods) & _HARD_BLOCK_LIFECYCLE_METHODS` for blocked lifecycle methods
       - Check `info.has_force_update` for forceUpdate usage
       - If blocked lifecycle methods found: return `ComponentClassification(file_path=info.file_path, component_name=info.component_name, tier=MigrationTier.MANUAL, rationale=f"Hard-blocked: contains {', '.join(sorted(blocked))}")`

       - If both: combine into one rationale string
       - If neither: return None
       - Google-style docstring

    4. Create `apply_heuristics(info: ComponentInfo) -> ComponentClassification`:
       - Count complexity signals: +1 for "componentDidUpdate" in lifecycle_methods, +1 for has_refs True, +1 for has_state True AND len(lifecycle_methods) > 2 (proxy for complex state usage)
       - signals == 0: tier = SAFE_AUTO, rationale = "No complexity signals: no componentDidUpdate, no refs, simple lifecycle"
       - signals <= 2: tier = ASSISTED, rationale listing which signals were found
       - signals > 2: tier = MANUAL, rationale listing all signals
       - Return ComponentClassification with computed tier and rationale
       - Google-style docstring

    5. Create `classify_component_deterministic(info: ComponentInfo) -> ComponentClassification`:
       - Call apply_hard_blocks first; if result is not None, return it
       - Otherwise call apply_heuristics and return result
       - This is the public entry point for deterministic classification
       - Google-style docstring
